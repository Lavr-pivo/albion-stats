# Albion Market Stats

Сайт статистики рынка Albion Online: цены по городам, история, поиск выгодных
перепродаж и (в планах) калькулятор крафта.

**Данные:** [The Albion Online Data Project](https://www.albion-online-data.com/)
— открытый публичный API. Сайт albiononline2d.com **не парсится** (он блокирует
ботов, и это не нужно — у AODP тот же первоисточник, легально и стабильно).

## Требования
- **Node.js ≥ 24** (используется встроенный `node:sqlite` без флага; в 22.x он
  экспериментальный и требует `--experimental-sqlite`).

## Запуск
```bash
npm install
npm run sync:items     # один раз: загрузить список предметов (RU-названия) в БД
npm start              # сервер + фоновый sync цен → http://localhost:3000
```
`npm start` сам подгрузит предметы, если БД пустая.

### Стратегия обновления цен
Чтобы укладываться в лимиты AODP (180/мин и 300 за 5 мин), синк двухуровневый:
- **горячий** — каждую минуту обновляет ~500 самых активных предметов (`HOT_SET_SIZE`);
- **полный** — раз в 30 мин обновляет весь каталог.

Запросы троттлятся (≈55/мин), что безопасно для лимита 300/5мин при постоянной работе.

### Переменные окружения
- `PORT` — порт (по умолчанию 3000)
- `DATA_DIR` — каталог для БД (на Railway укажи путь Volume, напр. `/data`)
- `NO_SYNC=1` — не запускать фоновый sync
- `SYNC_SERVERS=europe,west,east` — какие серверы синхронизировать
- `HOT_SYNC_INTERVAL_MS` — период горячего синка (по умолчанию 60000)
- `FULL_SYNC_INTERVAL_MS` — период полного синка (по умолчанию 1800000)
- `HOT_SET_SIZE` — размер горячего набора (по умолчанию 500)
- `AODP_MIN_INTERVAL_MS` — пауза между запросами к AODP (по умолчанию 1100)
- `SYNC_LIMIT` — ограничить число предметов в полном синке (для тестов)

Разовый синк цен без сервера: `npm run sync:prices` (можно с `SYNC_LIMIT`).

## Деплой на Railway
1. Запушить репозиторий на GitHub.
2. В Railway: **New Project → Deploy from GitHub repo**, выбрать репо.
3. (Рекомендуется) добавить **Volume** и смонтировать на `/data`, затем задать
   переменную `DATA_DIR=/data` — иначе БД сбрасывается при каждом редеплое.
4. Старт (`npm start`), healthcheck (`/healthz`) и Node 24 (`.nvmrc`) уже настроены
   в `railway.json` и `package.json`.

## Страницы
- `/` — цены по городам + поиск
- `/flips.html` — выгодные перепродажи (с учётом налога и комиссии)
- `/item.html?item=ID` — график истории цен
- `/craft.html` — калькулятор крафта (в разработке)

## API
- `GET /api/items?q=` — поиск предметов
- `GET /api/prices?item=&server=&cities=&qualities=` — текущие цены
- `GET /api/history?item=&server=&days=&scale=` — история
- `GET /api/flips?server=&quality=&maxAge=&minProfit=&maxRoi=&premium=` — перепродажи
- `GET /api/meta` — конфиг (серверы, города, размер БД)

## Заметки по данным
Данные краудсорсовые: возможны устаревшие цены и единичные «троллинг»-ордера
(например, выставленные за абсурдную сумму). Поэтому:
- везде показывается возраст данных (`observed_at`);
- flip-поиск отсекает устаревшее (фильтр свежести) и аномальный ROI (`maxRoi`).
Перед реальной сделкой проверяй цену в игре.
